Ingeniería del Software

¿Qué es la Ingeniería del Software?

La ingeniería del software es la actividad de crear programas de manera organizada, siguiendo pasos bien definidos. No se trata solo de escribir código: implica planificar, diseñar, programar, probar, lanzar y mantener el software de manera eficiente y segura.

Fases del Desarrollo de Software

1. Planificación e Investigación: Se define la idea, se evalúa la viabilidad técnica y comercial y se identifican los recursos necesarios.

2. Análisis de Requisitos: Se detallan todas las funcionalidades y características que debe cumplir el software, documentando necesidades del usuario y objetivos del proyecto.

3. Diseño y Prototipado: Se crea la arquitectura del sistema, la base de datos, y la interfaz de usuario mediante prototipos visuales o mockups para validar la experiencia de usuario.

4. Desarrollo o Implementación: Se programa el software siguiendo buenas prácticas y metodologías como SCRUM o Kanban. Se realiza control de versiones y revisiones de código.

5. Pruebas y Garantía de Calidad: Se ejecutan pruebas unitarias, de integración y funcionales para detectar errores o fallos antes del lanzamiento.

6. Despliegue o Implementación: Se publica la aplicación en el entorno de producción, tiendas de apps o servidores, asegurando que los usuarios puedan acceder correctamente.

7. Mantenimiento y Soporte: Se corrigen errores que aparecen en producción, se realizan actualizaciones de seguridad y se añaden mejoras según la retroalimentación de los usuarios.

8. Análisis y Optimización: Se analizan métricas de uso y rendimiento, para planificar mejoras y garantizar la eficiencia del software en versiones futuras.

Roles en la Ingeniería del Software

Arquitecto y Diseñador de Software: Planea la estructura general del proyecto, asegurando solidez y escalabilidad.

Desarrollador de Software: Implementa la aplicación y asegura que funcione según lo planeado.

Ingeniero de Integración y Pruebas: Comprueba que el sistema cumpla con los estándares de calidad antes del lanzamiento.

Analista de Sistemas / Consultor: Traduce las necesidades del cliente en requerimientos claros para el equipo de desarrollo.

Gestor de Proyectos: Coordina tiempos, recursos y asegura que el proyecto se complete según el plan.

Especialista en Mantenimiento: Gestiona actualizaciones, optimizaciones y corrección de errores post-lanzamiento.

Modelos de Ciclo de Vida del Software

1. Modelo en Cascada: Lineal y secuencial; cada fase debe completarse antes de la siguiente.

2. Modelo en V: Relaciona cada fase con pruebas específicas para mejorar la calidad.

3. Modelo Incremental: El software se desarrolla en módulos que se integran progresivamente.

4. Modelo Iterativo: Se realizan iteraciones con revisiones continuas y mejoras basadas en retroalimentación.

5. Modelo Espiral: Desarrollo iterativo centrado en la gestión de riesgos en cada ciclo.

6. Modelo Prototipado: Se crea un prototipo inicial para validar requisitos antes del desarrollo final.

7. Modelo DevOps: Integración continua, despliegue y monitorización automática de software.

RUP (Rational Unified Process)

Proceso iterativo basado en UML, organizado en fases:

1. Inicio (Inception): Definición de alcance, actores, requisitos principales y riesgos iniciales.

2. Elaboración (Elaboration): Detalle de requisitos, diseño de la arquitectura base y mitigación de riesgos críticos.

3. Construcción (Construction): Desarrollo de funcionalidades, pruebas internas y preparación del software para producción.

4. Transición (Transition): Formación a usuarios, ajuste final y entrega del producto.

Metodologías Ágiles

Basadas en flexibilidad, colaboración, iteraciones cortas y entregas frecuentes de software funcional.

Principios fundamentales:
• Entrega continua de software funcional
• Adaptación a cambios incluso en etapas avanzadas
• Comunicación constante con el cliente
• Equipos autoorganizados
• Mejora continua

SCRUM: Trabajo en sprints de 1 a 4 semanas. Roles: Product Owner, Scrum Master, Equipo.

KANBAN: Flujo continuo mediante tablero de tareas y control de trabajo en curso.

Extreme Programming (XP): Calidad de código mediante TDD, programación en parejas y refactorización continua.

Entornos de Desarrollo Integrado (IDE)

Características comunes:
Editor, autocompletado, depurador, compilador, integración con Git, gestor de errores, documentación integrada.

Características específicas:
Según el lenguaje o área: Java (refactorización), Python (consola interactiva), Web (vista previa), Móvil (emuladores).

Ventajas:
Mayor productividad, detección rápida de errores, integración de herramientas y control de versiones.

Inconvenientes:
Consumo de recursos, curva de aprendizaje inicial y posible dependencia de plugins.

Fases del Desarrollo de una Aplicación Informática

  • 1. Idea y Conceptualización: Definir el objetivo de la aplicación y su público.
  • 2. Planificación: Establecer requisitos, recursos, plazos y presupuesto.
  • 3. Diseño: Crear la arquitectura del sistema, UI/UX y prototipos.
  • 4. Desarrollo: Programación siguiendo estándares y metodologías de trabajo.
  • 5. Pruebas: Test unitarios, integrales y funcionales para asegurar calidad.
  • 6. Implementación: Lanzamiento de la aplicación en producción o tiendas.
  • 7. Mantenimiento: Corrección de errores, actualizaciones y nuevas funcionalidades.
  • 8. Optimización y Análisis: Evaluación del rendimiento, experiencia de usuario y mejoras futuras.

Máquinas Virtuales

Concepto: Una máquina virtual (VM) es un software que emula un computador completo dentro de otro, permitiendo ejecutar sistemas operativos y aplicaciones de forma aislada.

Efectividad: Permiten pruebas seguras, ejecutar múltiples sistemas en una sola máquina y aislar aplicaciones sin afectar al sistema principal.

Lenguajes de Programación

Clasificación:
• Bajo nivel: cercano al lenguaje máquina (ensamblador).
• Alto nivel: más cercano al lenguaje humano (Java, Python, C#).
• Interpretados: ejecutados línea por línea (Python, JavaScript).
• Compilados: convertidos en código máquina antes de ejecutar (C, C++).

Ventajas de los lenguajes de alto nivel:
• Más fáciles de aprender.
• Código más legible y mantenible.
• Desarrollo más rápido.

Desventajas de los lenguajes de alto nivel:
• Menor control sobre hardware.
• Pueden ser más lentos que los lenguajes de bajo nivel.

Elementos básicos:
• Variables
• Tipos de datos
• Operadores
• Estructuras de control (if, loops)
• Funciones o métodos
• Clases y objetos (en programación orientada a objetos)

Conceptos de código: Conjunto de instrucciones que indican a la computadora qué hacer, escritas en un lenguaje de programación.

Traductores:
• Compiladores: traducen todo el código fuente a lenguaje máquina antes de ejecutarlo.
• Intérpretes: traducen y ejecutan el código línea por línea durante la ejecución.
